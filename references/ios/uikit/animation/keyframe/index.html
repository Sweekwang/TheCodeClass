How to create a UIView Keyframes Animation?

UIView Keyframes - UIKit

Official Reference


Keyframe animation allows you to move to multiple location within a closure. This combine individual keyframs into a keyframes animation. Consider the example below:

The first stage of the ball animation accelerates which it take up 20% of the total time while it is moving upwards. The second stage, the ball speed slowly decreases which take up as 35% while the last tage takes up 35%.

The animation could be hard to create if the code is long. By breaking down into different stages, you can write your code easily.

Keyframes Syntax:

keyframe animation has two parts. The first part of the code is when you need set up the keyframe animation. You need to state the duration, or/and delay of the animation like what you did in UIView.animation. After that, you can add the keyframe into the closure inside the animations properties of the setting.

Setting keyframe animation

The syntax below is similar to the one in UIView.animate. You set the duration which is the total time of the keyframe (equals to 100% in the timeline shown above). Then you can add delay, options, animation which inside will be the keyframes and then completion (what will happen after the animation.)

Swift Code:

UIView.animateKeyframes(withDuration: TimeInterval, delay: TimeInterval, option: [ ], animations: {
    // Add the individual keyframes here
}, completion: nil)

Adding keyframes

After you have set up the animation keyframe, you can add individual keyframe inside the animation properties.

  • withRelativeStartTime - is the relative values of the start time in ratio. 0.0 is the start time while 1.0 is the end time.
  • relativeDuration - is the relative values of duration. How long will the animation. If the animation take up 40% of the total time, it will be 0.4.

Swift Code:

UIView.addKeyframe(withRelativeStartTime: TimeInterval, relativeDuration: TimeInterval) {

}

withRelativeStartTime and relativeDuration example

Using the diagram below is from the football animation, I will be showing you how to calculate the relative time.

The first frame of the animation start at a withRelativeStartTime of 0.0 and it takes 20% of the total time so relativeDuration is 2.0. Second frame starts after the first frame and so withRelativeStartTime is 0.0 + 2.0 (2.0) and relativeDuration is 0.35. Last frame has a withRelativeStartTime of 0.0 + 2.0 + 0.35 (0.55) and relativeDuration is 0.45.

If the duration (withDuration) is 5 seconds, relative time of 2.0 will be 1 seconds ( 5 * 0.2 ). Later on if you decided to change the total duration to a different value, the individual keyframe animations will recalculate accordingly.

Football example

The code below is an example of an animation show in the above example. A UIImageView is added to the storyboard without any constraint with a football images. The expected outcome is shown below:

Swift Code:

UIView.animateKeyframes(withDuration: 5, delay: 0, option: [ ], animations: {
    UIView.addKeyframe(withRelativeStartTime: 0.0, relativeDuration: 0.2) {
        self.ballImageView.center.x += 40
        self.ballImageView.center.y -= 30
        self.ballImageView.transform = CGAffineTransform(rotationAngle: CGFloat(Double.pi))
    }
    UIView.addKeyframe(withRelativeStartTime: 0.2, relativeDuration: 0.35) {
        self.ballImageView.center.x += 160
        self.ballImageView.center.y -= 120
        self.ballImageView.transform = CGAffineTransform(rotationAngle: CGFloat(0))
    }
    UIView.addKeyframe(withRelativeStartTime: 0.55, relativeDuration: 0.45) {
        self.ballImageView.center.x += 200
        self.ballImageView.center.y -= 180
        self.ballImageView.transform = CGAffineTransform(rotationAngle: CGFloat(Double.pi))
    }
}, completion: { _ in
    self.ballImageView.frame = CGRect( x: 10, y: 700, width: 100, height: 100 )
})
