How to create a UIView Animation?

UIView Animate

Official Reference


You are able to animate numerous of components using UIView.animate to create cools effects. With this knowledge you should be able to move and fade any components, changing animation ease and repeating an animation.

However, using this method only, you are not able to animate constraints. Do read at constraints animate topic if you want to animate constraints but you are still required to know the knowledge here before going into constraints animate topic.

The example used for the explaination below has a starting code of this (shown below). An UILabel is use to create a movement animation. This UILabel is added into the storyboard without any constraints.

Swift Code:

classViewController: UIViewController, UITextViewDelegate {
    @IBOutlet weak var label: UILabel!

    override func viewDidLoad() {
        super.viewDidLoad()

    }

}

Step 1: Before you start creating animation, you have to make sure that your component is at the initial position or state. For example, if I want move to a component from point A (out of the screen) to point B (original locations) as shown in the image below.

You can place the initial position (Point A) at viewWillAppear() because it postion the components before it appear. However, as mentioned this cannot be used if constraints are applied on the view.

The code below move the label out of the screen by subtracting the the width of the screen. In this way, we can ensure that the label is always out of the screen no matter the screen size.

Build and run your project to make sure your label truly disappear just as you had planned.

Swift Code:

override func viewWillAppear(_ animated: Bool) {
   super.viewWillAppear(animated)

   label.center.x -= view.frame.width

}

Step 2: Now you can animate the component (label) back to their original locations (point B) via animate method.

The simplest animation method take in duration and a closure as a input. The declaration is shown below.

  • withDuration : The duration for the animation
  • animations : Closure expression which state the final state of the components

Swift Code:

UIView.animate(withDuration: TimeInterval, animations: {

})

Inside the closure, add the position or the original state of the component and this method should be inside viewDidAppear().

Swift Code:

override func viewDidAppear(_ animated: Bool) {
   super.viewDidAppear(animated)

   UIView.animate(withDuration: 1, animations: {
      label.center.x += view.frame.width
   }
}

That's great you created an basic animation.

Adding Delay. If you have mutiple steps for an animation, or you have mutiple components to animate at a different timing then you should add delay to your animation.

Similary, we use the same animate method but it takes in a different parameters.

  • Delay : The amount of seconds will wait before the animation begin
  • Options : customization of the animation will explain more below.
  • Completion : Closure expression which the block of code run when the animation is finished.

Swift Code:

UIView.animate(withDuration: TimeInterval, delay: TimeInterval, options: [ ], animations: {

} , completion: { } )

Continue from the example above, after moving the label to point B. We shall move down the label. As shown below

We add a delay of 1 seconds and so the label move down after 1s.

Swift Code:

override func viewDidAppear(_ animated: Bool) {
   super.viewDidAppear(animated)

   UIView.animate(withDuration: 1, animations: {
      label.center.x += view.frame.width
   }

   UIView.animate(withDuration: 1, delay: 1, options: [ ], animations: {
      label.center.y += 100
   } , completion: nil )
}

Options. The options parameters allows you to modify the animation such as repeat (to replay the animation in an infinite loops)

Swift Code:

UIView.animate(withDuration: 1, delay: 1, options: [ .repeat ], animations: {
   label.center.y += 100
} , completion: nil )

You can chain more than one modification such as [.repeat, .autoreverse] by using comma to separate the two components. Autoreverse repeatedly plays your animation forward, then in reverse rather then forward direction only.

Lists of options:

Animation optionsMeaning
.repeatRepeat the animation forever
.autoreverseAnimation move forward and then backward when it repeat
.curveEaseInAnimation being it slowly and then speed up at the end
.curveEaseOutAnimation being it quickly and then speed down at the end
.curveEaseInOutCombination of curveEaseIn and curveEaseOut
