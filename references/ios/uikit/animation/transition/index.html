How to create a UIView Transition Animation?

UIView Transition - UIKit

Official Reference


Spring animation move to their locations, overshoot and then bounce back and finally, to the location. This create a bouncing animation which create a real-world feel.

Rather then using UIView.animate, transitions API is used to make various changes to make transition more naturally.

Prerequisites:

Before you start learning about spring animation, you have to know the following method and knowledge

  1. UIView.animate

Syntax and effects

Transition create an animation witin a container view which means that transition animate the container view by adding or removing any view from the subview while the animation runs.

To animate, you call a method similar to the one you did in UIView.animate. However, you can choose the predefined transition effects. The effects available are shown below.

  • with - is the container view.
  • option - is the animation options same as the one you seen in UIView.animate but there are more for transitions. See below to see more

Swift Code:

UIView.transition(with: UIView, duration: TimeInterval, option: [ ], animations: {

}, completion: { })

Transition effects

Removing and Adding

The transition animate the container view by adding or removing any view from the subview while the animation runs as shown in the image below. The container view will be still there but the view is added or removed.

Adding:

Swift Code:

let newView = UIView(frame: CGRect(x: 0, y: 0, width: 100, height: 100))
UIView.backgroundColor = .red

UIView.transition(with: containerView, duration: 1, option: [ .transitionCrossDissolve ], animations: {
    self.containerView.addSubview(newView)
}, completion: nil)

Removing:

Swift Code:

UIView.transition(with: containerView, duration: 1, option: [ .transitionCrossDissolve ], animations: {
    self.containerView.removeFromSuperview()
}, completion: { })

Appearing and Hiding

As show above, we only cover about adding or removing a view but you can also use transition to hide or show a container view. (right image is hidden but not color changes)



Swift Code:

UIView.transition(with: containerView, duration: 1, option: [ .transitionCrossDissolve ], animations: {
    self.containerView.alpha = 0
}, completion: {
    self.containerView.isHidden = true
})

Replacing

You can also change a view to another view.

Two news properties you seen here. From and to.

Swift Code:

UIView.transition(from: oldView, to: newView, duration: 1, option: [ .transitionFlipFromTop ], completion: nil)
