What is Optional in Swift?

Optional- iOS Swift

Official Reference

Variable or constants that you see it before contains a value such as Int number = 5. However, sometime, it can be empty or nil. For example, when asking a user about their age and it not required for us if they do not want to enter this information. Hence, when he/she will submit nil as a value.

How can we do it?

optional safety feature created by swift which can represent as a value or absense of value. Therefore, optional can hold a value or nil. For example, opional variable or constant is an box that can contain one value such as "Wes", 3 or 3.2 or empty. it is empty, it said to contain nil. An optional is referring to the box and not the item inside it.

Create a optional

You can create a optional by declaring a question mark after the data type. This means you create an box which can contain value or nil unlike the normal variable which contain only value.

Swift Code:

var age: Int?

After that you can assign it to a value or nil, if you did assign to anything, it is still empty (nil)

Swift Code:

age = 26
age = nil


Optionals

Now you want to looks what is inside the box. How will it looks like when you retrieve the information.

Swift Code:

var age:Int? = 26
print( age )


Output:

Optional(26)

The output is not what you wanted because the result is an optional that contains the value 26. If you take an opional as an normal integer to do a arithmetic operation. It will resulted in an error because you are trying to mutiple the optional or box and not the value inside.

Swift Code:

var age:Int? = 26
print( age * 3)


Output:

error: value of optional type 'Int?' must be unwrapped to a value of type 'Int'

Unwrapping optionals

The error message tells you that the optional have to be unwrapped. There are three ways to do that

  1. Force Unwrapping
  2. Optional binding
  3. Nil coalescing
  4. Unwrapping with guard

1. Force Unwrappings

You can force unwrapping it meaning you tell the Swift that is is an value inside and it is not nil. To wrap it, use exclamation mark after the variable name which tells the compiler that you want to look inside the box and take out the value.

Swift Code:

var age:Int? = 26
print( age! * 3)


Output:

78

However, you will get an error if you force unwrap an nil value.

Swift Code:

var age:Int? = nil
print( age * 3)


Output:

Fatal error: Unexpectedly found nil while unwrapping an Optional value: file

How can you play it safe?

A way to play it safe is to use an if statement and excute the code if it is not nil. However, this solved the problem but it is not perfect. As this method is tedious and sometime, you might forget it.

Swift Code:

var age:Int? = nil
if age != nil {
    print( age * 3)
}

2. Optional binding

A better way of doing it is to use a optional binding or if let which let you safety access the value inside the optional. In this method, there is no exclamation marks. If the value is not nil then the first block of code will be executes which you can safety use the variable inside the block of code.

If it is it nil, then another block of code is excuted.

Swift Code:

var age:Int? = nil
if let unwrappedAge = age {
    print( unwrappedAge)
} else {
    print( "My age is not included")
}


Output:

My age is not included

However, because it is hard to name a variable, it’s common practice to give the unwrapped variable the same name.

Swift Code:

if let age = age {
    print( age )
} else {
    print( "My age is not included")
}

3. Nil coalescing

Sometime you want to get a value out no matter what and in the case of nil, you will then assign it want a default value. To do that, you use a double question mark (??). When it is nil, it will assign it to the default value you stated of the right side of the double question mark.

Swift Code:

var age:Int? = nil
print( age ?? 0 * 3)


Output:

0

4. Guard

Guard is useful to use if you want to execute a function if the condition is true. The guard statement show below if the condition is false, the else block of code will be excuted and then it return to end the function.

Swift Code:

func getAge(age: Int?){
    guard let age = age else {
        return
    }
}
